## このマニュアルの目的

ハリーが作ったマキサロンのLP（ホームページ）を、**ハリーとマキの2人で共有して更新できる体制**にすることが目的です。

「共有運用」とは、こういうことです:

```
       GitHub（ソースコード保管庫・無料）
          │ Collaborator 2人で共有
          │
   ┌──────┴──────┐
   │              │
 ハリー         マキ
   │              │
   └──────┬──────┘
          ↓ どちらかが更新すると
          │
       Vercel（公開サーバー・無料）が自動で本番サイトに反映
          │
          ↓
   https://makisalon.vercel.app/
```

**ポイント**:
- ハリー側のGitHub・Vercelアカウントを軸に共有運用します
- マキは無料でGitHubにアカウントを作るだけ
- マキは GitHub Desktop と VS Code というアプリだけで作業します（黒い画面のコマンド不要）

## 共有方式の方針（重要・最初に押さえる）

| 領域 | 方式 | 補足 |
|------|------|------|
| **GitHub（ソースコード）** | Collaborator招待でマキにWrite権限付与 | 無料・両者編集可 |
| **Vercel（公開サーバー）** | ハリー側 Hobby プランで片側運用 | マキはVercel管理画面に入らない・git pushすれば自動で本番反映される仕組み |
| **本番反映の流れ** | マキ push → PR → ハリーレビュー → mainマージ → Vercel自動デプロイ | PR必須で事故防止 |
| **将来の完全移管** | repo所有権をマキに transfer → マキのVercelで新規接続 → カスタムドメイン | `05_完全移管手順.md` 参照 |

**なぜVercelは片側運用なのか**:
Vercel Hobbyプラン（無料）はTeam Memberの招待ができない仕様だからです。
ただし「GitHub経由のgit push自動デプロイ」は無料で動くので、マキが普通に作業して push すれば、本番サイトに反映されます。
マキは Vercel ダッシュボードを開く必要がないので、不便はほぼありません。

## できること / できないことの整理

### マキにできること（無料）
- ✅ LPのテキスト・画像・色の編集
- ✅ 編集内容を自分のPCで確認（プレビュー）
- ✅ PR（Pull Request）で「ここ変えました」をハリーに送る
- ✅ ハリーがOKしたら自動で本番反映

### マキができないこと（ハリーに依頼）
- ❌ Vercel管理画面（Analytics・Logs・環境変数の編集）
- ❌ 本番ドメイン設定変更
- ❌ 大規模なReactコードの新規開発（要相談）
