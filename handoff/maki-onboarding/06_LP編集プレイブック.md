## 2. 公開アセット（画像/動画）追加の正しい手順

### ❌ やってはいけない（本セッションのミス再現）
1. `public/images/` に画像ファイルを直接置く
2. `salon-content.ts` の配列を新ファイル名に変更
3. commit/push → 本番404
4. （Vite dev では public/ から直接配信されるので**ローカルでは動いて見える罠**）

### ✅ 正しい手順
```powershell
# 1. ファイルを public/images/ に配置
# 2. git status で untracked を確認
git status --short public/images/

# 3. 明示的に add（スペース入りファイル名は引用符必須）
git add "public/images/Instagram Gallery_1.png" "public/images/Instagram Gallery_2.jpg"

# 4. git ls-files で追跡されたことを確認（最重要）
git ls-files "public/images/Instagram*"

# 5. salon-content.ts 等の参照配列を更新

# 6. commit & push
```

### .gitignore の罠
本プロジェクトの `.gitignore` は以下の特殊ルールあり:
```
*.png                   ← png は全て無視
!public/**/*.png        ← public 配下の png だけ例外で追跡可
*.mjs                   ← mjs は全て無視
!vite.config.ts         ← 例外
!scripts/build-handoff-pptx.mjs ← 例外（手動追加分のみ）
*.cjs
!eslint.config.js
*.err                   ← dev-server.err 等
.vercel                 ← Vercel project.json 秘匿
```

→ **新しい .mjs スクリプトを scripts/ に置いても、明示的に `!` 例外を追加しないと追跡されない**

## 3. レスポンシブ余白の Rules of Thumb

### 縦並び要素のモバイル余白を計算する

flex/grid の `gap-X` と、内部コンポーネントの `mb-X` は**両方が累積する**。

```jsx
<div className="flex flex-col gap-16">   {/* 64px */}
  <ComponentA />                          {/* 内部 mb-14 = 56px */}
  <ComponentB />
</div>
```
→ A の下と B の上の間: **64 + 56 = 120px**（モバイル巨大空白の典型例）

### 対策パターン
| パターン | コード例 |
|---|---|
| **A. モバイルだけ縮小** | `gap-4 lg:gap-16` / `mb-2 lg:mb-14` |
| **B. コンポーネントに props化** | `<SectionHeading marginBottomClass="mb-2" />` |
| **C. ネガティブmargin（最終手段）** | `-mb-10`（保守性低い・推奨しない） |
| **D. min-h を レスポンシブ** | `h-[260px] md:h-auto md:min-h-[400px]` |

### 本セッションで使った具体例
- About.tsx 外側 flex: `gap-16` → `gap-4 lg:gap-16`
- About.tsx モバイル SectionHeading: `marginBottomClass="mb-2"` を渡す
- Company.tsx section: `py-24` → `py-16 md:py-24`
- Company.tsx 地図カラム: `min-h-[400px]` → `h-[260px] md:h-auto md:min-h-[400px]`

## 4. iframe の minHeight インラインstyle 罠

```jsx
<iframe
  style={{ minHeight: '400px', ... }}   ← ❌ メディアクエリ非対応
  ...
/>
```

React のインラインstyleは**メディアクエリ非対応**。モバイルでも400px固定で表示され、地図のロード失敗時には400px真っ白の空白になる。

### 対策
- インラインstyle の minHeight を**親要素のレスポンシブ class** に置き換える
- iframe の style は `style={{ border: 0, filter: '...' }}` 等の必要最小限のみ

## 5. 共通コンポーネント変更時の鉄則

`SectionHeading` のような複数箇所で使われるコンポーネントは、**最小公倍数の挙動**を default にする。

### ❌ ダメな変更
```diff
- <div className={`${textAlign} mb-14`}>
+ <div className={`${textAlign} mb-4`}>   {/* 全使用箇所が影響！ */}
```

### ✅ 正しい変更（後方互換）
```diff
  interface SectionHeadingProps {
    ...
+   marginBottomClass?: string;
  }

- const SectionHeading = ({...}) => {
+ const SectionHeading = ({..., marginBottomClass = 'mb-14'}) => {
    ...
-   <div className={`${textAlign} mb-14`}>
+   <div className={`${textAlign} ${marginBottomClass}`}>
```

→ 既存呼び出し（5箇所以上）は `marginBottomClass` 未指定で default `'mb-14'` のまま動作。

### 変更前に必ずやること
```powershell
# 1. 該当コンポーネントの全使用箇所を確認
# 2. 既存呼び出しに影響しない設計を確定
# 3. props に optional で追加（default は旧挙動）
```

## 7. モバイル/PC で異なるレイアウト構造を実現する 2 パターン

### A. CSS 一発分岐（軽量・推奨）
```jsx
<div className="flex flex-col-reverse lg:flex-row">
  <A />
  <B />
</div>
```
→ モバイルは B→A、PCは A→B の表示順

### B. 同一コンテンツを 2 箇所に描画（visibility切替）
本セッションの About モバイル見出し移動はこちら採用:
```jsx
{/* モバイル専用 */}
<motion.div className="lg:hidden">
  <SectionHeading ... />
</motion.div>

{/* PC専用 */}
<motion.div className="hidden lg:block">
  <SectionHeading ... />
</motion.div>
```

**B 採用の判断基準**: `flex-col-reverse` だと paragraphs まで上に行ってしまう場合、見出しだけ移動できる B を使う。

**注意点**: 同じ要素を2回描画するため、スクリーンリーダーが両方読み上げる可能性。必要なら片方に `aria-hidden="true"` を追加。

## 10. dev server / screenshot 撮影の Tips

### 本セッションで使った構成
- `screenshot.mjs`: Puppeteer ベース、モバイル（393x851 Pixel5）+ PC（1440x900）両方撮影
- 出力: `C:/Users/naoya/AppData/Local/Temp/maki_*.png`
- 注意: `.gitignore` で `*.mjs` 除外 → screenshot.mjs は untracked のまま運用

### モバイル/PC両方撮影の汎用テンプレ
```javascript
// モバイル
await page.setViewport({ width: 393, height: 851, isMobile: true, deviceScaleFactor: 2 });
await page.goto('http://localhost:5173', { waitUntil: 'networkidle0' });

// PC
await page.setViewport({ width: 1440, height: 900, isMobile: false, deviceScaleFactor: 1 });
await page.reload({ waitUntil: 'networkidle0' });
```

### scrollIntoView の罠
- `document.querySelector('#section-id')` で要素取得できても、画像遅延ロード等で scroll 位置がずれることがある
- 対策: `await new Promise(r => setTimeout(r, 2000));` で待機を入れる
- それでも届かない時は `window.scrollTo({ top: absolute-y })` の絶対座標指定

## 11. 電話番号など共通データの配置ルール

### 一元管理ファイル
- `src/config/company.ts` ... 会社情報（住所/電話/SNS）
- `src/config/salon-content.ts` ... コンテンツテキスト・画像配列

### 法的文書 vs 表示用の分離
本セッションで追加した設計:
```ts
// src/config/company.ts
phone: '053-472-3683',            // 法的文書用（固定電話）
directPhone: '08041820727',        // 直通モバイル
phoneDisplay: '053-472-3683\n直通：08041820727',  // Company/Footer 表示用
phoneNote: '※施術中は電話に出られません。留守番電話に...',
```

- `phone` → Law.tsx / Privacy.tsx で参照（法的文書として固定電話のみ）
- `phoneDisplay` → Company.tsx / Footer.tsx で参照（両方表示）
- `phoneNote` → 注意書き

→ **目的別にフィールド分離**することで、法的文書を巻き込まず表示変更可能。
