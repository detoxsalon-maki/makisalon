# 05. 完全移管手順（将来用・ハリー実施）

> 将来、LPの管理を完全にマキ側に移す時の手順です。
> このマニュアルは **ハリーが実施** することを前提に書かれています。
> 移管時期が来たら、マキとスケジュール調整してから実施してください。

---

## 完全移管とは

現状（過渡期）:
- GitHub: ハリーアカウント `harry-n2/makisalon`（マキは Collaborator）
- Vercel: ハリーHobbyアカウント・本番URL `makisalon.vercel.app`

完全移管後:
- GitHub: マキアカウント `<MAKI_USERNAME>/makisalon`（オーナー=マキ・ハリーは離脱 or Collaborator）
- Vercel: マキ Hobbyアカウント・本番URLは**カスタムドメイン**（例: `www.makisalon.jp`）

---

## 移管前のチェックリスト

実施前に以下を確認してください:

- [ ] マキが日々の更新フローに十分慣れている（最低3ヶ月運用後を推奨）
- [ ] マキがVercel管理画面の操作にも対応できる（または別のサポーターがいる）
- [ ] マキが独自ドメインを取得済み（または取得予定確定）
- [ ] 移管予定日のスケジュール調整完了
- [ ] 移管中の本番サイト停止時間（10〜30分程度）の許容確認

---

## 移管フロー全体像

```
Step 1: 事前準備（独自ドメイン取得・DNS確認）
   ↓
Step 2: GitHub repo の所有権を harry-n2 → <MAKI_USERNAME> に transfer
   ↓
Step 3: マキの Vercel アカウントから新規プロジェクトとして接続
   ↓
Step 4: カスタムドメインをマキ Vercel 側で設定
   ↓
Step 5: 旧 Vercel プロジェクト（ハリー側）を削除 or アーカイブ
   ↓
Step 6: 動作確認・マキへの最終引き渡し
```

## Step 1: 事前準備

### 1-1. マキが独自ドメイン取得

マキが以下のいずれかでドメイン取得（ハリーが代理でもOK）:
- お名前.com
- ムームードメイン
- Google Domains（現Squarespace Domains）

例: `www.makisalon.jp` `makisalon-tokyo.com` 等

### 1-2. ドメイン管理画面のアクセス権

ドメインのDNS設定をマキが触れるようにしておく（マキのアカウントで登録、ハリーは管理者として補助）。

### 1-3. バックアップ作成

念のため、現状の最終 main を別ブランチ・別フォルダに保管:

```
# ハリー側ローカル PowerShell
cd "C:\Users\naoya\myproject\エステサロン_改善版\maki-luxury-lp"
git branch backup/before-transfer-YYYY-MM-DD
git push origin backup/before-transfer-YYYY-MM-DD
```

ローカルフォルダごとデスクトップに `makisalon_backup_YYYYMMDD` でコピー貼り付けも実施。

## Step 2: GitHub repo を Transfer

GitHubの公式 Transfer Ownership 機能を使います。**履歴・PR・Issue全部そのまま引き継がれます**。

### 2-1. Transfer ownership 実行（ハリー実施）

1. https://github.com/harry-n2/makisalon/settings を開く
2. ページ最下部の **Danger Zone** セクションへ
3. **`Transfer ownership`** ボタンをクリック

   `[📸スクショ位置: Danger Zone の Transfer ボタン]`

4. ダイアログ:
   - **Repository name to confirm**: `harry-n2/makisalon`
   - **New owner's GitHub username**: `<MAKI_USERNAME>`
5. **`I understand, transfer this repository`** をクリック
6. パスワード認証 / 2FA認証

### 2-2. マキ側で受領（マキ実施）

マキのメールに「Transfer notification」が届きます。**3日以内** に Accept しないとキャンセルされます。

1. メール内の **`Accept transfer`** をクリック
2. または GitHub 通知センター（右上ベル）から Accept
3. Accept すると repo が `<MAKI_USERNAME>/makisalon` に変わる

### 2-3. ハリーを Collaborator として残すか判断

マキ側で **Settings → Collaborators** に行き、ハリー（`harry-n2`）を Collaborator として追加するか判断:
- ✅ 残す: ハリーが今後もサポート可能（メンテ・改修対応）
- ❌ 外す: 完全独立

### 2-4. ローカル設定の更新（マキ・ハリー両方）

clone済みフォルダのリモートURLが古いままなので、両者更新する:

**GitHub Desktop で更新**:
1. GitHub Desktop → `Repository` → `Repository settings...`
2. **`Remote`** タブ → **Primary remote repository (origin)** の URL を新URL（`https://github.com/<MAKI_USERNAME>/makisalon.git`）に変更
3. **Save**

または PowerShell で:
```powershell
cd "(makisalonフォルダのパス)"
git remote set-url origin https://github.com/<MAKI_USERNAME>/makisalon.git
git remote -v   # 確認
```

## Step 3: マキの Vercel アカウントで新規プロジェクト接続

### 3-1. マキのVercelアカウント作成（無料）

1. https://vercel.com/signup
2. **`Continue with GitHub`** で Step 2 で移管された GitHub アカウントでサインアップ
3. プラン選択: **Hobby（無料）** を選ぶ

### 3-2. リポジトリをインポート

1. Vercel ダッシュボード → **`Add New...`** → **`Project`**
2. **Import Git Repository** で `<MAKI_USERNAME>/makisalon` を選択 → **`Import`**

   `[📸スクショ位置: Vercel Import 画面]`

3. プロジェクト設定:
   - **Project Name**: `makisalon` （好きな名前で）
   - **Framework Preset**: `Vite` を選択
   - **Build Command**: `npm run build`（自動検出されているはず）
   - **Output Directory**: `dist`（同上）
   - **Install Command**: `npm install`
4. **`Deploy`** ボタン

数分でビルド・デプロイ完了。新URL（例: `makisalon-xxxxx.vercel.app`）でアクセスできるようになります。

### 3-3. PR Preview の確認

Vercel ダッシュボード → Settings → Git → **Preview Deployments** を「All branches」または「Only production branch + PRs」に設定。
これでマキ側でも PR で自動プレビューURLが生成されるようになります。

---

## Step 4: カスタムドメイン設定

### 4-1. Vercel にドメイン追加

1. マキの Vercel プロジェクトを開く → **Settings** → **Domains**
2. ドメインを入力（例: `www.makisalon.jp`） → **`Add`**

   `[📸スクショ位置: Domains 設定画面]`

3. Vercel が要求するDNS設定を確認:
   - 一般的に **CNAME**: `cname.vercel-dns.com.`
   - apex（`www`なし）の場合は **A record**: `76.76.21.21`

### 4-2. ドメイン管理画面でDNS設定

ドメイン取得時のレジストラ（お名前.com 等）の管理画面で、DNS設定を変更:

| Type | Name | Value | TTL |
|------|------|-------|-----|
| CNAME | www | cname.vercel-dns.com. | 3600 |
| A | @ (apex) | 76.76.21.21 | 3600 |

設定後、反映に最大48時間（通常10分〜数時間）かかります。

### 4-3. 反映確認

Vercel の Domains 画面で、ドメイン横に **Valid Configuration** と表示されればOK。
ブラウザでドメインを開いてサイトが見えれば成功。

### 4-4. SSL証明書

Vercel が自動で Let's Encrypt の SSL を発行・更新します（無料・自動）。何もする必要なし。

## Step 5: 旧 Vercel プロジェクトの処理

カスタムドメインで動き始めたら、旧 `makisalon.vercel.app`（ハリー側）の処理:

### Plan A: 完全削除（推奨・割り切り）

1. ハリーの Vercel ダッシュボード → `beautysalon` プロジェクト → **Settings** → **Advanced**
2. ページ最下部 **Delete Project**
3. プロジェクト名を入力して削除

これで `makisalon.vercel.app` は使えなくなります。
旧URLでアクセスしてきたユーザーはエラーページになる点に注意。

### Plan B: リダイレクト設定（事前にユーザーが旧URLでブックマークしている場合）

旧プロジェクトを残し、新ドメインへリダイレクトする `vercel.json` を追加:

```json
{
  "redirects": [
    {
      "source": "/(.*)",
      "destination": "https://www.makisalon.jp/$1",
      "permanent": true
    }
  ]
}
```

→ コミットして旧プロジェクトに反映 → 一定期間（3〜6ヶ月）後に Plan A で削除。

### Plan C: そのまま放置

技術的には可能だが、両方のサイトが見える状態になるので推奨しない。

## トラブルシューティング

### Q. Transfer Ownership ボタンが押せない / 失敗する

考えられる原因:
- マキの GitHub username が間違っている
- マキの GitHub アカウントがメール認証を完了していない
- マキの GitHub プラン制限（無料Plan でも問題ないはずだが、Organizations が絡む場合は確認）

### Q. DNS設定したのにドメインが反映されない

- TTL 3600秒 = 1時間反映待ち。最大48時間
- `nslookup www.makisalon.jp` でDNS反映を確認
- お名前.com 等で「ネームサーバー」が他社のものになっていないか確認（独自DNSサーバーを使っている場合は変更）

### Q. 移管後にハリーが手伝う必要がある場合

- マキ側 repo の Collaborator として残ってあれば、PR レビュー・直接 push 可能
- 外れている場合は再度 Collaborator 招待を依頼
