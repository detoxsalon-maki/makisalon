## Step 1: マキをリポジトリに Collaborator として招待

### 1-1. リポジトリの Settings を開く

1. ブラウザで以下を開く:
   👉 https://github.com/harry-n2/makisalon

2. 画面上部のタブから **`Settings`**（一番右）をクリック

   `[📸スクショ位置: Settings タブのハイライト]`

### 1-2. Collaborators メニューへ

1. 左サイドバーの **`Collaborators`**（Access セクション内）をクリック
2. パスワード確認を求められたら、ハリーのGitHubパスワードを入力

   `[📸スクショ位置: Collaborators メニュー画面]`

### 1-3. マキを招待

1. **`Add people`** ボタンを押す
2. 入力欄にマキの GitHub username（`<MAKI_USERNAME>`）を入力
3. 候補に出てきたマキのアカウントを選択
4. **`Add <MAKI_USERNAME> to this repository`** ボタンを押す

   `[📸スクショ位置: Add people ダイアログ]`

### 1-4. 招待メールが自動送信される

マキの登録メールアドレスに「You've been added to harry-n2/makisalon」というメールが届きます。
マキ側でこのメール内の **`View invitation`** → **`Accept invitation`** を押すと参加完了です。

## Step 2: ブランチ保護ルール（PR必須）を設定

**目的**: マキ（とハリー）が誤って main ブランチに直接 push できないようにする。
これによって、必ず PR → レビュー → マージという流れになり、事故を防げます。

### 2-1. Branches 設定を開く

1. リポジトリの **Settings** → 左サイドバー **`Branches`**

   `[📸スクショ位置: Branches メニュー]`

### 2-2. ルール追加

1. **`Add branch ruleset`** または **`Add classic branch protection rule`** をクリック
   - 新しいUIなら "Add branch ruleset"
   - 旧UIなら "Add classic branch protection rule"

2. 設定項目（新UIの場合）:

   | 項目 | 設定値 |
   |------|--------|
   | Ruleset Name | `main-protection` |
   | Enforcement status | **Active** |
   | Target branches | **Add target** → **Include default branch**（= main） |
   | Bypass list | 空欄（ハリーも例外なくPR必須にする推奨） |

3. **Branch rules** の以下にチェック:
   - ✅ **Require a pull request before merging**
     - Required approvals: `1`
   - ✅ **Require linear history**（推奨・コミット履歴が綺麗になる）
   - ✅ **Block force pushes**

   `[📸スクショ位置: Branch rules チェックボックス]`

4. **`Create`** ボタンで保存

### 2-3. 確認テスト（推奨）

```
ローカルで main にいる状態で何か小さな変更をして git push しようとすると、
GitHub側で「Protected branch update failed」と弾かれることを確認
```

## Step 3: Vercel 側の確認（変更は基本なし）

### 3-1. 自動デプロイが効いていることを確認

1. https://vercel.com/dashboard を開く
2. `beautysalon` プロジェクト（makisalon.vercel.app のもの）をクリック
3. **Settings** → **Git** を開く
4. 以下を確認:
   - Production Branch: `main`
   - Connected repository: `harry-n2/makisalon`
   - Deploy Hooks: 任意（このマニュアルでは未使用）

   `[📸スクショ位置: Vercel Git設定画面]`

### 3-2. Preview Deployment が PR で自動生成されることを確認

`Settings` → `Git` → **Preview Deployments** が **All branches** または **Only production branch + PRs** になっていればOK。
これで、マキが PR を出すたびに自動で `https://makisalon-git-xxxxx-...vercel.app` というプレビューURLが生成されます。

### 3-3. マキを Vercel Member には招待しません

理由: Hobbyプラン（無料）の規約上、個人利用のみで他人を Member 招待することは商用利用扱いになり規約違反になるため。
マキは Vercel管理画面に入らずとも、git push するだけで本番反映されるので問題ありません。

## トラブルシューティング

### Q. マキが招待メールを受け取っていない

1. マキに GitHub にログインしてもらい、右上アイコン → **`Your organizations`** または直接 https://github.com/notifications を確認
2. それでも見つからない場合、ハリー側で `Settings` → `Collaborators` → マキの行に **Pending invitation** と表示されていれば、横の **Copy invite link** で招待リンクを取得してマキに直接送る

### Q. ブランチ保護を設定すると自分（ハリー）も困る

それで正解です。ハリーも PR レビューフローに乗ることで、事故を完全に防げます。
緊急時は一時的にルールを無効化することもできます（同じBranches設定画面で `Enforcement status` を `Evaluate` に変更）。

### Q. Vercel ダッシュボードでデプロイが失敗している

1. プロジェクトの **`Deployments`** タブで失敗したデプロイをクリック
2. **Build Logs** を確認
3. 大抵は TypeScript エラー or 依存パッケージのバージョン不整合
4. 原因不明な場合はハリー側ローカルで `npm install && npm run build` を試してエラー再現を確認
